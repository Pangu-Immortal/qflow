## README 沉淀规范

工程根目录有且仅有一个 README.md，每次功能完成后检查并更新：

```
# 项目名称
## 项目概览（做什么/不做什么 等）
## 架构设计（模块职责/设计决策 等）
## 工程结构（目录说明）
## 运行环境（系统/版本/依赖/硬件配置要求 等）
## 从零搭建指南（面向初次接触工程的小白：环境准备/依赖安装/部署步骤/常用命令）
## 快速启动（环境检查/启动/调试/常见问题）
## 核心流程列表（主流程/关键分支 等）
## 技术债与风险（已知问题/不建议修改的区域 等）
```
