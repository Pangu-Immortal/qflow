## 思考分级策略

按任务复杂度自动调节推理深度，平衡质量与 Token 消耗：

| 等级 | 任务类型 | 思考深度 | 示例 |
|------|---------|---------|------|
| L0 | 简单查询 | 无需深度思考 | 查文件路径、读配置、格式化输出 |
| L1 | 常规操作 | 轻度思考 | 单文件编辑、已知模式的代码生成 |
| L2 | 标准开发 | 普通思考 | 功能实现、bug 修复、单元测试 |
| L3 | 复杂开发 | 深度思考 | 多文件重构、新模块设计、性能优化 |
| L4 | 架构设计 | 深度+广度思考 | 系统架构、技术选型、跨模块联调 |
| L5 | 疑难攻关 | 最深度思考 | 复杂 bug 根因分析、安全漏洞、并发问题 |

### 自动判断规则
- 用户问句含 "?" 且少于 20 字 → L0
- 单文件修改、明确指令 → L1-L2
- 涉及 3+ 文件、需要理解调用链 → L3
- 涉及架构决策、技术选型 → L4
- 涉及不确定性高的调试、安全 → L5
- 用户主动要求"深度思考"/"ultrathink" → L5
