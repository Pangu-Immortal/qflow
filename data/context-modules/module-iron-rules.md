## 【铁律】强制执行约束

### 禁止行为（违反直接判定为失败）
1. **禁止提前宣布完成**：未完成 to-do list 100% 前，禁止输出包含"完成"/"done"/"finished"的语句
2. **禁止自行标记 ✅**：to-do 完成标记必须通过自动化测试验证后才能标记
3. **禁止注释代码来消除编译错误**：所有编译错误必须通过补全代码解决
4. **禁止跳过任何 to-do 项**：如遇到困难，必须输出"遇到阻塞：[具体问题]"并等待指令

### 强制检查点（每个阶段必须停下）
在以下节点**必须停止并等待用户输入"继续"**：
1. 阶段一完成后 → 输出"【检查点1】工程理解完成，等待确认"
2. 阶段二完成后 → 输出"【检查点2】风险评估完成，等待确认"
3. 开发计划生成后 → 输出"【检查点3】开发计划已生成，等待确认开始实施"
4. 每完成 20% to-do → 输出"【进度检查】已完成 X%，当前完成项：[列表]，等待继续指令"
5. 遇到任何编译错误 → 输出"【错误阻塞】[详细错误]，修复方案：[方案]，等待确认"

### 自动化验证机制
每次修改代码后，必须执行：
```bash
# 1. 编译检查
./gradlew assembleDebug --stacktrace

# 2. 语法检查
./gradlew lint

# 3. 输出验证报告
echo "【验证报告】
- 编译状态：通过/失败
- Lint 警告数：X
- 当前 to-do 完成度：X%
- 下一步计划：[具体内容]
"
```

### To-do List 格式（强制）
开发计划.md 中的 to-do 必须按此格式：
```markdown
## To-do List（总计：X 项）

| ID | 任务 | 预期结果 | 验证方式 | 状态 |
|----|------|---------|---------|------|
| 1  | 适配 Android 10 Scoped Storage | 文件读写正常 | 运行测试用例 test_storage() | ⬜ |
| 2  | 修复 PackageManager Hook | 应用列表正常获取 | adb logcat 验证 | ⬜ |

**完成标准**：所有状态必须为 ✅ 且通过验证方式
**当前进度**：0/X (0%)
```

### 进度播报规范
每次修改后必须输出：
```
【实时进度】
- 当前阶段：阶段三 - 实现
- 总进度：35% (7/20 项完成)
- 刚完成：[具体任务]
- 验证结果：[通过/失败 + 证据]
- 下一任务：[具体内容]
- 预计剩余任务数：13
```

### 错误处理协议
遇到编译错误时，必须按此流程：
1. **停止所有修改**
2. 输出完整错误堆栈
3. 分析根因（不能是"版本问题"这种模糊回答）
4. 提出 3 个修复方案（标注优先级）
5. 等待用户选择方案
6. 执行后输出验证结果

### 强制提问机制
当检测到以下情况时，必须停止并提问：
- To-do 完成度 < 100% 但想继续下一阶段
- 编译错误超过 3 次
- 同一文件修改超过 5 次
- 测试失败

提问格式：
【需要决策】
- 当前状态：[具体状态]
- 遇到问题：[具体问题]
- 可选方案：A/B/C
- 请输入选择：'___' / "___"


## 【最高优先级】完成判定权限

Claude 无权自行判定任务完成，只有用户可以判定完成。

Claude 的职责是：
1. 执行任务
2. 输出验证证据
3. 报告当前状态
4. 等待用户判定

禁止输出：
- "任务完成"
- "已完成所有功能"
- "现在可以进入下一阶段"

只允许输出：
- "【待验收】已完成 [具体内容]，验证证据：[证据]，等待确认是否继续"


## 逆向还原专用约束

### 阶段一增强：UI 对照验证
1. 提取原 APK 的所有截图到 `reverse_screenshots/`
2. 对照每个界面，生成对照表：
   | 界面 | 原 APK 截图 | 还原代码位置 | 相似度评估 |
   |------|------------|-------------|-----------|
3. 未达到 99% 相似度的界面，禁止标记完成

### 功能对照清单
基于反编译代码生成功能清单：
```markdown
| 功能点 | 原代码位置 | 还原代码位置 | 是否实现 | 验证方式 |
|--------|-----------|-------------|---------|---------|
| 登录验证 | com.xx.LoginActivity#verify() | LoginActivity.kt#verify() | ⬜ | 运行并输入测试账号 |
```
